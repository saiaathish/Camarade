# Camarade product site

An original one-page product narrative for [Camarade](https://github.com/saiaathish/Camarade), built around its public Stage 1 specification: context audit, evidence-backed task contracts, and matched baseline comparison.

The site is deliberately honest about the current product state. It includes no install command, runtime screenshot, or benchmark claim because those do not yet exist in the public repository.

## Run locally

```bash
npm install
npm run dev
```

Open `http://localhost:5173/`.

## Verify

```bash
npm run typecheck
npm run build
npm run qa:visual
```

The visual QA command expects the dev server at `http://127.0.0.1:5173/`. It checks desktop, tablet, 200%-zoom-equivalent, mobile, narrow-mobile, keyboard, reduced-motion, link, console, SVG, overflow, interaction-race, axe-core accessibility states, and every dashboard route and fixture state. Captures and the machine-readable report are written to `.artifacts/qa/`.

## Dashboard `/runs/`

The dashboard lists recorded runs at `/runs/` and one run per page at `/runs/<comparison-id>/`. It is fixture-backed in S8-03: every value comes from the canonical Stage 8 fixtures, and every page carries the notice "Simulated fixture data — not benchmark evidence."

- Contract: `src/dashboard/contract.ts` at the repository root is the read-only source of truth (`stage-8-dashboard.v1`). The frontend display types in `src/dashboard/dashboard-types.ts` mirror it.
- Data source: components only use the `DashboardDataSource` interface (`listRuns`, `getRun`). `FixtureDashboardDataSource` is the S8-03 implementation; S8-04 swaps the factory in `src/dashboard/fixture-dashboard-data-source.ts` for a local API data source. No HTTP, filesystem, or backend code runs in the browser.
- Fixtures: `scripts/generate-dashboard-fixtures.mjs` reads the canonical root fixtures and emits `src/generated/dashboard-fixtures.ts` (git-ignored, regenerated by `dev`, `build`, and `typecheck`). A missing fixture, a wrong schema version, or a duplicate comparison ID fails the build.
- Empty-list mode: `/runs/?fixture=empty` renders the canonical empty-run-list fixture state.
- Safe IDs: comparison IDs must match `^[A-Za-z0-9][A-Za-z0-9._:-]{0,119}$`. Unknown or unsafe IDs render a safe not-found state with no fixture data.
