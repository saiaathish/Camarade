# Camarade website copy

## Copy rule

Every heading must make sense by itself. Do not write a bold claim followed by a paragraph that translates it. Use context, compression, code changes, tests, and saved files instead of internal contract language.

## Home `/`

- Headline: Camarade gives your coding agent only the context it needs.
- Actions: See how it works / View on GitHub

### Product proof

- Heading: Camarade compresses messy context into a clear task.
- Scroll story: One synthetic, noisy task request contains misspellings, irrelevant contact information, stale paths, an unnecessary dependency, and unrelated work. As the visitor scrolls, those fragments collapse out of the same prompt and one concise task remains. No visible status labels or field-by-field explanation interrupt the transformation.

## Compiler `/compiler/`

- Headline: Camarade checks your repository context against the code and keeps only what matters for the task.
- Proof: Messy context becomes compressed context.
- Inputs: `AGENTS.md`, `CLAUDE.md`, `.cursor/rules/**`, `.github/copilot-instructions.md`, `README.md + docs/**`, and relevant code + config.

## Compare `/experiment/`

- Headline: Camarade runs the same coding task with the original context and the compressed context.
- Method: Same code. Same model. Same tests. Different context.
- Approval: You approve the setup before either run starts.

## Evidence `/evidence/`

- Headline: Camarade saves the context, code changes, tests, and run settings from both runs.
- Integrity boundary: Camarade records the results. It does not choose a winner or create a score.
